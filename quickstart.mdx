---
title: "Quickstart: set up your first Git hook with Lefthook"
description: "Write a lefthook.yml config, install hooks into your Git repository, and run your first automated pre-commit check in under five minutes."
---

Lefthook turns a single YAML file into a full Git hooks setup. This guide walks you through writing your first configuration, installing the hooks, and seeing them run on a real commit.

<Steps>
  <Step title="Write your configuration">
    Create a `lefthook.yml` file in the root of your Git repository. The example below runs a CSS linter and a JavaScript linter in parallel on every `pre-commit`, passing only the staged files that match each glob pattern. The `stage_fixed: true` option automatically stages any files the linter auto-fixes.

    ```yaml lefthook.yml
    pre-commit:
      parallel: true
      jobs:
        - name: stylelint
          run: yarn run stylelint --fix {staged_files}
          glob: "*.css"
          stage_fixed: true

        - name: eslint
          run: yarn run eslint --fix "{staged_files}"
          glob:
            - "*.ts"
            - "*.js"
            - "*.tsx"
            - "*.jsx"
          stage_fixed: true
    ```

    You can add as many jobs as you need. Jobs run in parallel by default when `parallel: true` is set, so your hooks finish faster than running them sequentially.
  </Step>

  <Step title="Install the hooks">
    Run `lefthook install` once in your repository root. Lefthook writes scripts into `.git/hooks/` that call `lefthook run` automatically for each configured hook.

    ```bash
    lefthook install
    ```

    You should see output similar to:

    ```
    sync hooks: pre-commit ✓
    ```

    Commit this `lefthook.yml` to your repository so that every teammate gets the same hooks when they run `lefthook install` after cloning.
  </Step>

  <Step title="Make a commit">
    Stage some files and commit. Lefthook intercepts the `pre-commit` hook and runs your jobs automatically.

    ```bash
    git add -A && git commit -m "feat: add new feature"
    ```

    Lefthook prints the result of each job. If a job fails, the commit is blocked and you see the output from the failing command. Fix the issues and commit again.

    You can also trigger any hook group manually without making a commit:

    ```bash
    lefthook run pre-commit
    ```

    This is useful for testing your configuration before pushing it to the team.
  </Step>
</Steps>

## Next steps

<Tip>
Create a `lefthook-local.yml` file alongside `lefthook.yml` to override or extend your team's configuration on your local machine. Add it to `.gitignore` so personal overrides stay private. For example, you can skip jobs you don't need locally or point commands at a different binary path.

```yaml lefthook-local.yml
pre-commit:
  jobs:
    - name: eslint
      skip: true
```
</Tip>
