Lefthook is a fast, powerful Git hooks manager written in Go. It lets you configure all your pre-commit checks, linters, and custom scripts in a single `lefthook.yml` file, then installs them into your repository's `.git/hooks/` directory with one command. Whether you work with Node.js, Ruby, Python, or any other language, Lefthook runs in any environment without additional runtime dependencies.

## Why Lefthook?

Lefthook is built for teams that want reliable, fast Git hooks without the overhead of language-specific hook managers.

## How it works

<Steps>
  <Step title="Create your configuration">
    Add a `lefthook.yml` file to your project root and define which commands to run for each Git hook.

    ```yaml lefthook.yml
    pre-commit:
      parallel: true
      jobs:
        - run: yarn eslint --fix "{staged_files}"
          glob: "*.{js,ts,jsx,tsx}"
          stage_fixed: true

        - run: bundle exec rubocop --force-exclusion -- {staged_files}
          glob: "*.rb"
    ```
  </Step>
  <Step title="Install the hooks">
    Run `lefthook install` once. Lefthook writes scripts to `.git/hooks/` that call `lefthook run` automatically on each Git operation.

    ```bash
    lefthook install
    ```
  </Step>
  <Step title="Commit with confidence">
    Your hooks now run automatically. Lefthook checks your staged files, runs jobs in parallel, and blocks the commit if anything fails.

    ```bash
    git add -A && git commit -m "feat: add new feature"
    ```
  </Step>
</Steps>
